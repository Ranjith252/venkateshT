## Local build

Install dependencies and build the production bundle:

```bash
npm install
npm run build
```

The production build output will be written to `dist/`.

## Vercel deployment

Recommended Vercel project settings:

- Build Command: `npm run build`
- Output Directory: `dist`
- Framework Preset: `Vite` (optional)

If your repo is connected to Vercel and the project is set to build from `main`, Vercel will run the `npm run build` command and deploy the contents of `dist/`.

## Environment variables

This project does not require server-side environment variables for core frontend features. If you previously used a backend SMTP service, it has been removed from the repository.

If you integrate a backend later, set any server env variables in the Vercel Dashboard under the project Settings → Environment Variables.

## Notifications & email

- The app includes in-app notifications (BroadcastChannel + desktop notifications) that work without a backend.
- A simple SMTP backend scaffold was previously present but removed. If you want server-side email delivery, re-add a backend and configure SMTP credentials in the deployment environment.
